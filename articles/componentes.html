Componentes em Angular

ANGULAR

Os componentes são os blocos básicos da construção de interface em Frameworks Front-End. Sua utilização faz com que se tenham blocos de código reutilizáveis, tornando o HTML principal mais conciso e de fácil leitura, além de encapsular a complexidade do componente, facilitando sua manutenção e modularizar a aplicação, auxiliando a padronização do desenvolvimento. O Angular, tendo uma arquitetura baseada em componentes, utiliza desta conceito para estruturar e organizar aplicações.

Em Angular, um componente é composto de 4 partes principais. São elas:

  • Seletor: seria o "nome" do componente, ou seja, a forma como ele será instanciado no documento HTML principal.
  • Template: seria a estrutura HTML do componente. Esta estrutura pode ser repassada de forma inline, com a propriedade template ou repassando um arquivo HTML do componente, com a propriedade templateUrl.
  • Folha de estilo: seria o código CSS (ou SCSS) de estilização da estrutura que foi repassada no template. Da mesma forma que o template, pode ser passado de forma inline. Em geral, se repassa um ou mais arquivos com a extensão escolhida (.css ou .scss) dentro de duplo colchetes, como um array ([]), como valor da propriedade styleUrls.
  • Classe: onde são definidas as propriedades e comportamentos do componente.

Quando uma aplicação Angular é inicializada, um componente principal chamado app-root, a "raiz" da aplicação, é criado. Todos os demais componentes irão ser componentes descendentes deste componente principal. Para criar um novo componente, o Angular possui o comando ng generate component nome-componente em que nome-componente é um nome arbitrário. É possível, da mesma forma, utilizar o atalho ng g c nome-componente. Por padrão, quando este comando é utilizado, é criada uma pasta com o nome do componente indicado contendo 4 arquivos:

  • nome-componente.component.ts, o arquivo do componente com a extensão da linguagem TypeScript.
  • nome-componente.component.html, o arquivo de template HTML do componente que utiliza uma sintaxe extendida, suportando diretivas do Angular.
  • nome-componente.component.css, o arquivo de estilização do componente.
  • nome-componente.component.spec.ts, o arquivo de especificações de testes do componente.

No arquivo do componente, desenvolvido em TypeScript, é possível conectar todas as outras partes principais do componente de modo a definí-lo por completo. Sua estrutura inicia com a importação do pacote Component do Angular, além de quaisquer outras funcionalidades a serem utilizadas; segue com o decorator @Component onde serão inseridos o seletor, o template e a folha de estilo e finaliza com uma classe que irá conter suas propriedades e comportamentos. Dentro desta classe, temos como padrão uma função construtora e o método ngOnInit, que contém o código que é aplicado quando o componente é inicializado.

Além de criada a pasta com os arquivos mencionados anteriormente, o módulo principal da aplicação é atualizado, importando o componente e o declarando, de modo a possibilitar a utilização deste componente dentro da estrutura prncipal.

Portanto, com sua inicialização, definição e importação, é possível utilizar componentes para suas funções principais na aplicação: servir como blocos para a construção da interface e controlar a sua view, definida pelo template, que renderiza o que o usuário final irá visualizar e com o que ele irá interagir.
