Media Queries

RESPONSIVIDADE

As media queries são recursos do CSS em que, mediante o valor verdadeiro de uma ou mais condições, um bloco de código de estilização será aplicado no documento. Utilizado principalmente para adaptar o layout de uma aplicação à diferentes tamanhos de tela, as media queries são um dos 3 principais pilares do Design Responsivo.

A estrutura de uma media query é sempre iniciada pela regra @media. Logo após, os seguintes elementos completam a sintaxe:

  • media type: é o tipo de mídia, que é opcional e pode assumir os valores all, que serve para todos os tipos de mídia e é implícito caso não se indique um media type, print, que serve para visão de documentos ou material paginado visualizados em modo de impressão, ou screen, que representa telas de qualquer dispositivo. Outros media types existiam em versões anteriores do media queries, mas foram depreciadas no Media Queries 4, que é a versão mais atual.
  • media features: são as condições as quais o dispositivo deve obedecer para que o bloco de código CSS seja aplicado. As opções de media features são muitas, mas as mais utilizadas no design responsivo são min-width, max-width, orientation e aspect-ratio.
  • código CSS: incluso dentro de chaves, fica o código CSS que será aplicado caso a media query retorne verdadeira. Este código é sempre carregado, porém apenas aplicado mediante a condição descrita anteriormente.

Para um conjunto de media features ou regras mais específicas, é possível utilizar os seguintes operadores lógicos na construção das condições:

  • and: operador lógico de adição, une um media type definido às media features ou combina múltiplas media features como em @media screen and (min-width: 768px) and (max-width: 1279px). Para que o código CSS seja aplicado, todas as condições conectadas por and devem ser verdadeiras.
  • not: operador de negação, pode negar tanto uma media type, uma media feature ou toda a media query. Utilizando o operador not, expressões que retornariam falso, retornam verdadeiro. Com seu uso, a inclusão de um media type se torna obrigatória.
  • only: aplica o estilo apenas se todas as condições da media query forem atendidas. Em browsers antigos, sem a palavra reservada only, caso a primeira parte da media query fosse verdadeira, o estilo já era aplicado sem considerar outras media features que poderiam ser falsas.
  • , (vírugla): este operador é utilizado para converter várias condicionais em media queries em uma só de forma que, se pelo menos uma das condições retornar verdadeiro, o bloco de código será aplicado.
  • or: adicionado na versão mais recente do media queries, é uma nova maneira de converter várias condicionais em uma só. Equivalente ao operador , (vírgula).

Um recurso que foi também adicionado na versão mais recente do media queries, objetivando deixar o código mais enxuto, é a sintaxe de range. Os operadores <, >, => e <= podem ser utilizados em qualquer media feature nas quais ranges podem ser aplicados. Os códigos exibidos em ambas as imagens a seguir possuem o mesmo resultado, porém o código com range é menos verboso.

Como exemplo de como o resultado das media queries são visualizadas, temos a própria página inicial deste blog com 3 diferenças principais entre elas: o menu de navegação, a barra de pesquisa e a quantidade de cards de artigos por linha.

Para a criação destas diferentes visualizações, foi criado um mixin em SCSS com as media queries necessárias para cada breakpoint (mobile, tablet e full HD) e, para cada ponto do site mencionado anteriormente, as devidas alterações nas propriedades de estilização foram aplicadas. O site do CSS blog foi criado com a abordagem desktop-first, portanto, não há uma media query para o tamanho em desktop, visto que a estilização inicial foi feita para este tamanho de dispositivo.

Em tamanhos de tela menores, o menu de navegação ficaria muito apertado ou de difícil leitura, portanto, foi criado um novo menu de navegação em que os tópicos seriam exibidos em coluna a partir do clique em um botão. Em telas maiores, o menu por meio do botão é escondido e o menu original fica em evidência. Em telas menores, o contrário é verdadeiro. Estas alterações são realizadas por meio da modificação na propriedade display dos menus nas media queries necessárias. Como o menu original é o "comportamento" default, não foi preciso incluir uma media query para telas em full HD.

Em telas de tamanho mobile, o tamanho da barra de pesquisa geraria um efeito indesejável de criação de uma barra de scroll horizontal no site. Desta forma, com a modificação da propriedade width na media query para dispositivos mobile, foi possível evitar este efeito.

Quanto menor o tamanho da tela, menor a quantidade de cards de artigo que ficariam legíveis e agradavelmente dispostos por fila da tela inicial. Diante disto, a modificação do grid template da listagem de artigos por meio da propriedade grid-template-columns foi essencial para que, de acordo com o tamanho do dispositivo, os cards não ficassem nem ilegíveis, nem grandes demais.

Portanto, com a aplicação das media queries descritas, foi possível adequar o layout de pontos da tela inicial que, sem a responsividade, tornariam a experiência do usuário mais dificultosa e menos agradável.
