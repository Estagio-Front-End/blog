Package Managers em Javascript

JAVASCRIPT

Package managers, em português, gerenciadores de pacotes, são ferramentas que auxiliam a instalação de pacotes ou módulos no sistema, facilitando o compartilhamento de código entre desenvolvedores e o reuso destes códigos. Estes pacotes ou módulos, em geral, são bibliotecas desenvolvidas para contribuir com um desenvolvimento mais rápido por possuírem funções e solucionarem problemas específicos. Em Javascript, os dois gerenciadores mais utilizados são o npm e o yarn.

npm

Como instalar o npm

O npm, ou Node Package Manager, é o gerenciador atrelado ao NodeJS. Sua instalação é realizada por meio da instalação do próprio NodeJS, em que o npm é também instalado como um "pacote adicional". Assim que a instalação do NodeJS termina, é possível verificar, por meio do terminal de comando, se a instalação do npm foi concluída com sucesso. Utilizando os comandos npm -v ou npm --version, caso o terminal retorne a versão do npm instalada e não gere erros, ele foi instalado corretamente. No terminal de comando, digitando npm ou npm help, é possível visualizar todos os comandos disponíveis para o npm.

Por meio do site npmjs.com é possível buscar e verificar informações de instalação e uso dos pacotes e módulos como express, swiper e axios.

Como inicializar o npm na aplicação

Digitando o comando npm init no terminal, dentro da sua aplicação, um arquivo chamado package.json é automaticamente gerado. Este arquivo contém todos os pacotes e suas versões cuja aplicação utiliza. Deste modo, movendo a aplicação ou realizando seu deploy, é possível sinalizar quais pacotes o projeto usa. Após digitar o comando mencionado, algumas perguntas acerca do projeto como nome do projeto, versão, autor, licença, entre outras, irão ser realizadas. Respondidas as perguntas, o terminal irá mostrar um resumo das respostas e perguntará se está tudo certo para continuar. Digitando yes, o arquivo package.json aparecerá no diretório do projeto.

Como instalar e desinstalar pacotes

Feito este processo, é possível instalar pacotes e salvá-los como dependências no arquivo package.json. Para tal, é só procurar o pacote que se deseja instalar no npmjs.com e verificar o comando de instalação no lado direito da tela. Após isto, é só digitar o comando no terminal dentro da pasta da sua aplicação, lembrando de incluir --save após o comando. Isto garantirá que o pacote será salvo no arquivo package.json. Em geral, para instalar pacotes o comando é npm i pacote ou npm install pacote (lembrando do --save). Para desinstalar pacotes, em geral, utiliza-se npm uninstall pacote, npm un pacote, npm remove pacote ou npm rm pacote (também utilizando o --save, para salvar o package.json sem a dependência).

Ao instalar um pacote pela primeira vez, a pasta node_modules e o arquivo package-lock.json são automaticamente criados. A pasta node_modules conterá todo o código do pacote instalado e o arquivo package-lock.json terá informações sobre o pacote para que, entre outros propósitos, desenvolvedores em projetos compartilhados possam instalar as exatas mesmas dependências. O package-lock.json será atualizado em toda operação que modificar a pasta node_modules ou o arquivo package.json.

Ao instalar dependências que serão utilizadas apenas no desenvolvimento e não serão necessárias em produção, utiliza-se o --save-dev após o comando de instalação. Deste modo, a dependência será uma dev dependency, e será identificada no package.json como tal.

Para instalar dependências em uma versão específica, em geral, temos o comando npm install pacote@versão. Porém, é indicado sempre procurar e confirmar, no npmjs.com ou na documentação da própria biblioteca, o modo instalação.

Para instalar dependências globalmente, de forma que os pacotes sejam instalados na própria máquina e possam ser utilizados em qualquer aplicação, o comando, em geral, é npm install -g pacote, sendo o -g o indicativo da instalação global. Para remover um pacote que foi instalado globalmente, deve-se utilizar o comando npm remove -g pacote.

Pontos positivos do uso de --save

Uma das grandes razões para o uso do --save ou --save-dev é para o caso em que algum desenvolvedor clone o seu projeto sem a pasta node_modules, mas com o arquivo package.json com todas as dependências utilizadas, é só digitar o comando node install que os pacotes utilizados serão automaticamente instalados. Sem o --save e sem as dependências no package.json, o comando só geraria um erro e a aplicação não poderia rodar normalmente.

Scripts

Dentro do arquivo package.json existe um objeto entitulado "scripts". Estes scripts são "atalhos" para realizar ações como rodar a aplicação, testar a aplicação, entre outras. Um atalho muito comum é o "start" para rodar a aplicação, em que se indica o arquivo .js principal da aplicação.
