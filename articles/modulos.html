Módulos em Angular

ANGULAR

Módulos em Angular são uma ferramenta de encapsulamento de código baseado em unidades coesas para cada função dentro da aplicação. Os módulos atuam como containers de componentes, diretivas e serviços, entre outros blocos de código, que atuam em um mesmo objetivo e/ou funcionalidade.

Este conceito auxilia na modularização da aplicação, gerando containers funcionais e facilmente reutilizáveis, além de facilitar a manutenção do código, tornando a aplicação mais facilmente testável e escalável.

Para criar um módulo, basta utilizar o comando ng generate module nome-modulo, em que nome-modulo é um nome arbitrário. Este comando gerará um arquivo com a extensão de TypeScript com o nome nome-modulo.module.ts. Até o Angular v16, aplicações criadas com o comando ng new geravam um arquivo app.module.ts por padrão, em que se poderia importar todos os componentes, diretivas e serviços, gerando um container que encapsula a aplicação como um todo. A partir do Angular v17, o padrão se tornou os componentes ditos "standalones", que não possuem módulos atrelados e podem ser utilizados na aplicação ou aninhados em outros componentes por meio de importação direta na propriedade imports dos componentes.

A estrutura inicial de um módulo consiste em suas importações seguido de um decorator @NgModule com as propriedades do módulo e uma classe exportando este módulo.

As propriedades mais importantes do decorador são a declarations, que é um array com os componentes e diretivas utilizados dentro daquele módulo; o imports, um array com quaisquer outros módulos utilizados naquele módulo; o providers, um array com os services utilizados naquele módulo e o bootstrap, que é o componente principal da aplicação (em geral, o AppComponent criado por padrão pelo Angular CLI).

Caso se queira reutilizar os componentes de um módulo em outro componente que não faça parte da descendência original dele, é possível exportar o componente com a propriedade exports no decorator @NgModules e importar o módulo original no módulo do componente em que se quer utilizar o componente original.

Nas imagens anteriores, é possível ver o trâmite de export/import entre módulos para utilização de um componente fora do escopo do seu módulo. Desta forma:

  • - Criando o ProductsModule
  • - Declarando e exportando o ProductItemComponent no decorator @NgModule do ProductsModule
  • - Importando o ProductsModule, por meio do import no arquivo e no decorator @NgModule, no AppModule
  • - O AppModule tendo o AppComponent declarado no seu decorator @NgModule

é possível utilizar o ProductItemComponent dentro do AppComponent.

Existem, também, módulos nativos do Angular que podem ser importados para utilizações específicas, como o BrowserModule, para renderização da aplicação em um navegador; o RouterModule, para criação de rotas da aplicação; o HttpClientModule, para comunicação com o servidor por meio do protocolo HTTP; o CommonModule, para utilizar diretivas como NgIf e NgFor; o ReactiveFormsModule, para construção de formulários reativos, entre outros.

Portanto, criando, importando e utilizando módulos, é possível deixar a aplicação mais concisa e organizada na troca de informações e componentes entre funcionalidades, além de facilitar a manutenção e os testes por toda a aplicação, aumentando sua escalabilidade.
