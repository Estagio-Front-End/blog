Display em CSS

POSICIONAMENTO

A propriedade display especifica com qual comportamento um elemento será renderizado na página. Alguns dos valores que essa propriedade pode assumir são:

  • inline: com este valor, o item será renderizado como uma caixa dentro de uma linha. Não há quebras de linha antes e após o item, de forma que, se houver espaço, vários itens inline estarão em uma mesma linha.
  • block: com este valor, o item será renderizado como uma caixa de bloco, em que há quebras de linha tanto antes quanto depois do conteúdo.
  • inline-block: este valor gera um item em bloco que será renderizado como um item único em uma linha.
  • flex: com este valor, o item se comporta como um container no modelo flexbox.
  • inline-flex: este valor gera um container do modelo flexbox dentro de um contexto de linha.
  • grid:com este valor, o item se comporta como um container no modelo grid.
  • inline-grid: este valor gera um container do modelo grid dentro de um contexto de linha
  • flow-root: este valor cria um novo contexto de formatação em bloco, ou seja, o item age como a tag html em termos de criação de um fluxo de layout do zero.
  • none: este valor faz com que o elemento seja renderizado como se não existisse, ou seja, o fluxo do documento não "guarda" o espaço reservado a ele. Caso se queira apenas reduzir a visibilidade, mas ainda mantendo o item no fluxo, deve-se utilizar as propriedades opacity ou visibility.

Também há outros valores para esta propriedade, como list-item, que mostra o item como se fosse uma tag li do HTML e table e seus valores internos, que fazem com que os itens sejam renderizados e se comportem como elementos de tabelas HTML.

Na seção de resultados da documentação MDN é possível testar como cada valor afetará a renderização dos itens.
