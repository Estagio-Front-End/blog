Animations

ANIMAÇÕES

As animations, assim como as transitions, também são mudanças de estado de elementos. As principais diferenças entre esses dois tipos de mudanças são que as animations não necessariamente precisam de um gatilho para acontecerem e podem acontecer infinitamente, e que as animations permitem um controle muito maior dos estados intermediários entre o início e o fim da animação.

Para esse controle seja efetivo, é utilizada a regra @keyframes, que permite que os estados de transição seja definidos com porcentagens (%) ou com palavras reservadas como from (estado inicial de 0%) e to (estado final de 100%).

Para aplicar as animações, é necessário, no bloco de estilização do item, indicar os valores das propriedades de animação. São elas:

  • animation-name: o nome da animação. Notar que este nome deverá ser o mesmo identificador do @keyframes correspondente.
  • animation-duration: o tempo de duração que a animação terá.
  • animation-timing-function: como a animação progride ao longo do seu ciclo. Pode receber valores como linear, ease-in-out ou steps(), entre outros.
  • animation-delay: tempo que a animação demorará para iniciar.
  • animation-iteration-count: número de vezes que a animação irá ocorrer, podendo receber valores numéricos ou infinite.
  • animation-direction: direção em que a animação vai correr, sendo ela normal, reversa, alternada ou alternada e reversa.
  • animation-fill-mode: define como a animação irá aplicar os estilos aos elementos antes e após a execução da animação. Pode receber valores como forward, backwards, entre outros.
  • animation-play-state: indica se a animação está correndo ou pausada. Recebe apenas o valores running ou paused.

Para manter o código conciso, existe a shorthand animation, que engloba todas as propriedades supracitadas em uma linha única. O shorthand mais comum para animações utilizado agrupa name, duration, timing function e iteration count, mas as demais propriedades listadas podem ser utilizadas na mesma linha de código CSS.

Para animações múltiplas, é possível utilizar uma shorthand para cada uma ou utilizar as propriedades originais separadamente e criar "listas" em cada uma, de modo que cada posição da lista em cada propriedade refletirá os valores de uma animação.
