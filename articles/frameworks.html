Frameworks Javascript

JAVASCRIPT

Frameworks no front-end são um conjunto de ferramentas desenvolvidas para facilitar o trabalho de pessoas desenvolvedoras na construção de aplicações. Essa facilitação ocorre por meio do fornecimento de estruturas de trabalho consistentes e conjuntos de recursos pré definidos. O uso de frameworks pode tornar o desenvolvimento mais rápido, o código mais consistente e padronizado e as aplicações mais escaláveis. Suas funcionalidades também podem ajudar no desenvolvimento de aplicações mais complexas com menos dificuldade em relação à utilização de Javascript puro.

Existem atualmente muitos frameworks e bibliotecas que visam facilitar o desenvolvimento de aplicações, mas os mais utilizados no desenvolvimento front-end são o Angular, o React e o Vue.

Angular

O Angular é um framework desenvolvido pelo Google e lançado em 2010. Em sua documentação, se denomina como uma plataforma de desenvolvimento que inclui um framework baseado em componentes, uma coleção de bibliotecas bem integradas e um ferramental para auxiliar desenvolvedores a desenvolver, construir, testar e atualizar suas aplicações.

Os principais recursos do Angular são a utilização da arquitetura MVC, a injeção de dependências, o two-way data binding e o desenvolvimento com TypeScript, uma linguagem derivada do Javascript, porém fortemente tipada.

Por ser um framework completo, uma das grandes vantagens de se utilizar Angular são os recursos disponíveis para a construção da aplicação, além de criar aplicações escaláveis e possuir uma comunidade ativa para suporte. A falta de flexibilidade do Angular pode ser uma vantagem ou desvantagem, a depender do ponto de vista, mas o ponto negativo mais expressivo do Angular é a maior curva de aprendizado em relação à outros frameworks e bibliotecas disponíveis.

React

O React é uma biblioteca desenvolvida pelo Facebook e lançada em 2011. Baseado em componentes, o React permite o desenvolvimento de grande parte de uma aplicação, mas não necessáriamente toda. A própria documentação indica que, caso se necessite construir uma aplicação completa, se use um framework fullstack como o Next.js ou o Remix.

Os principais recursos do React são a utilização do virtual DOM, a sintaxe em JSX, uma linguagem que permite a utilização de HTML e JS juntos no código, o one-way data binding e a possibilidade de criação de aplicativos mobile com React Native.

As grandes vantagens do React são a maior facilidade no aprendizado em comparação à outros frameworks ou bibliotecas e a grande comunidade desenvolvedora que utiliza esta biblioteca, tornando o suporte e a documentação muito mais vivos. Contrário ao Angular, o React é bastante flexível, o que pode ser uma vantagem ou desvantagem. Por outro lado, as grandes desvantagens do React são a necessidade de outras bibliotecas para complementar no desenvolvimento de aplicações mais robustas e a alta frequência de atualização, o que torna o acompanhamento por parte do desenvolvedor mais desafiadora.

Vue

O Vue é um framework desenvolvido por Evan You, ex funcionário do Google, e lançado em 2014. Ele se denomina em sua documentação como um framework acessível, performativo e versátil para o desenvolvimento de aplicações. Este framework possui uma abordagem progressiva, que permite que os desenvolvedores adotem gradualmente o Vue em suas aplicações.

Os principais recursos do Vue são suas bibliotecas para rotas e gerenciamento de estados, o gerenciamento de eventos, a possibilidade de criar componentes modulares e two-way data binding.

As grandes vantagens do Vue são seu fácil aprendizado para desenvolvedores que já são familiares com HTML, CSS e JS, além de ter um alto desempenho e ser escalável. Suas desvantagens, porém, são a menor comunidade em relação à outros frameworks como Angular e React e a grande quantidade de recursos em chinês, o que pode limitar a quantidade de recursos para desenvolvedores que desconhecem este idioma.

Angular x React x Vue

As diferenças entre estes frameworks, além de suas sintaxes, são variadas. Deste modo, é necessário um bom conhecimento da aplicação a ser desenvolvida e das suas necessidades para escolher qual framework ou biblioteca melhor se aplica. A seguir, alguns comparativos entre as três opções citadas anteriormente:

  • Renderização: as 3 ferramentas possuem a renderização pelo lado do cliente (CSR) como default, mas suportam renderização pelo lado do servidor (SSR) com o suporte de outras ferramentas como Angular Universal para Angular, Gatsby ou NextJS para React e Express ou NextJS para Vue.
  • Arquitetura: O Angular possui uma arquitetura baseada em MVC, mas não segue estritamente esta arquitetura visto que ele também é baseado em componentes. O React é completamente baseado em componentes e utiliza o virtual DOM para melhorar a performance da aplicação. O Vue segue uma arquitetura parcialmente baseada em MVVM, mais focada na camada ViewModel.
  • Ecosistema: o Angular possui o NgRx para gerenciamento de estados e o Angular Material para dar suporte em relação desenvolvimento baseado no Material Design,além de faze parte da stack MEAN. O React, além de fazer parte da stack MERN, possui no seu ecossistema o React Native, utilizado para o desenvolvimento de aplicações mobile. O Vue possui como uma de suas principais bibliotecas o Vuex, utilizado para gerenciamento de estados.
  • Ferramentas: o Angular possui ferramentas como o ng2-charts para gráficos, o videoangular para necessidades de vídeos na aplicação e o ng-animate para animações e transições. O React possui ferramentas como o Zustand para gerenciamento de estados com menor alocação de memória, o React Hook Form para facilitar a manutenção e validação de formulários e o react belle para componentes como cards, botões, entre outros. O Vue possui ferramentas como o Vue Router para rotas, o vuelidate para validação de formulários e o VueX para gerenciamento de estados.
  • Performance: Por utilizarem o virtual DOM em vez da árvore de DOM original, Vue e React possuem uma performance superior ao Angular. Por ser uma biblioteca, de modo que é mais leve que um framework, o React possui uma performance ligeiramente melhor que o Vue.
  • Curva de aprendizado: Por ser um framework progressivo, a curva de aprendizado de Vue é menor que a de React, que, por sua vez, é menor que a de Angular.

Desta forma, é possível ver que, para aplicações em larga escala que requeiram uma melhor estrutura, Angular é uma solução mais completa. Para aplicações de página única, Vue é um ótimo framework e, para interfaces baseadas em componentes e mais flexíveis, React é uma ótima escolha.
