Interfaces em Angular

ANGULAR

Interfaces em Angular são como contratos que firmam a estrutura de um objeto, suas propriedades e métodos juntamente com seus tipos, mas não fornece uma implementação. Nos casos de instanciação de um objeto com o tipo da interface ou extensão da interface por uma classe, é que serão definidos os valores de propriedades e implementações de métodos. As interfaces deixam o código mais legível, mais consistente e facilitam a manutenção de partes da aplicação nas quais são utilizadas, visto que o contrato define quais e os tipos de dados esperados como input ou resposta dentro de uma funcionalidade.

Para criar uma interface em Angular, basta utilizar o comando ng generate interface nome-interface em que nome-interface é um nome arbitrário, por convenção, iniciado com I (de interface), para diferenciação. Existe, também, o atalho ng g i nome-interface. Com a utilização deste comando, é criado um arquivo com a extensão de TypeScript (.ts) quase vazio, exceto pela estrutura inicial de uma interface, já sendo exportada.

Com a interface criada, é possível definir as propriedades e métodos na qual a interface irá consistir com seus respectivos tipos.

Propriedades com o símbolo ? são propriedades opcionais dentro do contrato estabelecido pela interface. É importante frisar que, propriedades que não estiverem dentro do contrato e forem utilizadas gerarão erro em tempo de compilação, assim como propriedades que estiverem dentro do contrato e não forem utilizadas. Propriedades com a palavra reservada readonly não poderão ser modificadas após instanciação da interface.

Para utilizar a interface criada, é necessário importá-la no componente que se deseja utilizá-la e aplicá-la.

Na construção de constante exibida na imagem anterior, até que todas as partes do contrato firmado com a interface fossem inseridas no objeto, a IDE continuava gerando erro por falta de propriedades. Desta forma, criando, importando e utilizando interfaces criadas, é possível atingir o objetivo esperado com esta ferramenta que é padronizar e dar mais previsibilidade à inputs ou respostas dentro da aplicação, de modo a identificar e corrigir erros mais rapidamente.
