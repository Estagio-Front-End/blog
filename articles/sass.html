SASS

CSS BÁSICO

O SASS é um pré processador CSS, sendo atualmente um dos mais utilizados, que permite adicionar funcionalidades ao código de estilização a fim de otimizar o código em termos de concisão, manutenibilidade e reutilização. Hoje, a sintaxe mais utilizada do SASS é a SCSS, que se assemelha bastante ao CSS puro, gerando menos esforço de adequação do código à sintaxe do pré processador. As funcionalidades, adicionais ou melhoradas, mais utilizadas são os imports, as variáveis, os encadeamentos, mixins e funções.

Imports

Semelhante ao CSS puro, recursos dentro da própria pasta do projeto ou recursos externos podem ser importados na folha de estilo SCSS. Muito utilizada para organização de código e importação de fontes externas, a sintaxe de importação é exibida nas imagens a seguir.

Para organização do código, é comum a estilização ser criada por componente em um arquivo "parcial" que depois é importado em uma folha de estilo global, que será pré processada. Para imports dessa natureza, o arquivo parcial deve conter um underline no início do nome, por exemplo, _componente.scss. No arquivo global, cujo nome não possui underline, ele é importado no formato da primeira imagem exibida anteriormente, apenas com o titulo do arquivo, no caso do exemplo: @import "componente";.

Variáveis

As variáveis em SCSS são mais facilmente criadas e utilizadas quando em comparação ao CSS puro. Para tal, basta colocar um cifrão ($), o nome da variável e atribuir seu valor. Para utilização da variável, é necessário apenas escrever o nome da variável com o cifrão no início como valor da propriedade a ser utilizada. Cores, fontes e dimensões que se repetem muito ao longo do código podem ser transformadas em variáveis para melhor manutenibilidade do código.

Encadeamentos

Apesar de já existir em versões mais atuais do CSS puro, os encadeamentos (ou aninhamentos) são muito mais concisos no SCSS. Para o encadeamento de seletores acontecer, basta que, dentro do par de chaves do seletor inicial, crie-se o bloco de código do seletor secundário. É indicado que, para melhor legibilidade do código, sejam feitos encadeamentos de, no máximo, 3 níveis.

Com o encadeamento utilizado na figura anterior, foi possível criar a estilização que, no CSS puro, seria criada em 4 blocos de códigos diferentes: .listagem-artigos__card a, .listagem-artigos__card a h2, .listagem-artigos__card a >img e .listagem-artigos__card span.

Mixins

Mixins são formas mais organizadas de criar blocos de código que serão repetidos várias vezes no código. Em vez de repetir as mesmas propriedades várias vezes ao longo do código, cria-se um mixin com as propriedades utilizadas e, em uma linha apenas, se inclui as propriedades no bloco de código do seletor desejado.

Para a criação de um mixin, é necessário adicionar @mixin, o nome pelo qual o mixin será identificado, os argumentos entre parêntesis (se existirem) e o par de chaves para delimitar o bloco de código. Se não houverem argumentos a serem passados, os parêntesis podem ser omitidos.

A regra @content é utilizada quando se tem estruturas que se repetem, porém conteúdos diferentes. Muito utilizada em mixins de media queries e animações, faz com que o código fique mais conciso sem perder sua adaptabilidade ao que se deseja realizar.

Para a utilização de um mixin, é necessário apenas incluir, no bloco de código do seletor escolhido, a regra @include juntamente com o nome do mixin e passar os argumentos necessários, se existirem.

Os mixins também são muito utilizados para incluir os chamados vendor prefixes, que são os prefixos para compatibilidade de certas propriedades em diferentes navegadores. O valor sticky, da propriedade position, é um exemplo de valor que necessita desses prefixos para operar bem em todos os navegadores. Deste modo, é possível criar um mixin que possua as propriedades com seus valores compatíveis utilizando os prefixos -webkit-, -moz-, -ms- e -o-, além da propriedade original. Na utilização da position: sticky, em vez de ser necessário escrever 5 linhas com todos os prefixos, se utiliza apenas uma para incluir o mixin.

Funções

Assim como em linguagens de programação, no SCSS, funções são blocos de código que são executados e, neste contexto, possuem um retorno obrigatório. A criação de uma função inicia com a regra @function, o nome da função, os argumentos que ela receberá entre parêntesis, as chaves delimitando o bloco de código e, dentro das chaves, a regra @return indicando o retorno final da função.

É possível passar mais de um argumento, desde que estes sejam separados por vírgula. Para argumentos que originalmente possuem vírgula, mas devem ser reconhecidos como apenas um argumento, como sombras diferentes em uma mesma propriedade box-shadow, é necessário adicionar ... após o nome do argumento que deve ter este comportamento. Desta forma, o argumento na assinatura da função é escrito como ($argumentos...) e os valores de argumento separados por vírgula na chamada da função serão interpretados como um só argumento.

A utilização de uma função se dá pela chamada do seu nome, com os argumentos a serem passados entre parêntesis, na propriedade em que se deseja utilizar o valor de retorno da função.

O SASS também possui módulos com funções nativas que podem ser utilizadas pelo desenvolvedor. Alguns dos módulos mais importantes são:

  • math module: com funções que operam números como ceil(), floor(), round(), max() e abs(), além de funções para potência, raiz quadrada e funções trigonométricas.
  • string module: possui funções utilizadas para combinar, separar ou modificar elementos do tipo string como insert(), length(), slice() e to-upper-case().
  • color module: possui funções utilizadas para obter informações de ou manipular cores como red(), green(), blue() e adjust().
  • list module: possui funções para manipulação de listas como append(), slice() e nth().

Também são módulos do SASS o map module, selector module e meta module. A documentação oficial traz todos os módulos existentes, assim como todas as funções inclusas e explicações de como utilizá-las.
