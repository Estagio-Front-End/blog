API em Javascript

JAVASCRIPT

As APIs, no português, Interface de Programação de Aplicações, são um mecanismo pelo qual diferentes aplicações trocam informações. APIs são utilizadas para acesso à dados de terceiros, abstrair a complexidade de uma aplicação, aumentar as funcionalidades de uma aplicação ou adicionar uma camada de segurança em dispositivos móveis. Aplicativos nativos de previsão do tempo, widgets de aplicativos e o sistema operacional perguntando se um app pode utilizar a localização atual são exemplos de APIs trabalhando.

Existem algumas formas de trabalhar com APIs, "consumindo-as" na aplicação e utilizando os dados ou funcionalidades obtidas no código. Exemplos são: fetch API, Axios e Ajax.

Fetch API

A Fetch API é uma API que possibilita a criação de requisições HTTP do navegador para o servidor. É uma evolução do XMLHttpRequest por ser mais simples e mais flexível. Essa API utiliza, dentro de uma estrutura de Javascript assíncrono (async/await ou .then), o método global fetch() para inicializar uma requisição. Ela é baseada em Promise, que são, como o nome sugere, um "promessa" de retorno de dados. A depender do resultado da Promise, pode-se lidar com erros utilizando o bloco catch ou acessar informações da resposta (Response) como o seu valor, o status da resposta e o valor do status da resposta.

Para métodos de requisição que requeiram o envio de dados, o método fetch() é passível de receber um segundo argumento, o objeto init, por onde são passadas as informações da requisição, incluindo os dados a serem enviados para o servidor.

AJAX

O AJAX é uma junção entre Javascript assíncrono e XML (ou JSON, ou txt) para enviar e receber dados entre cliente e servidor. Uma requisição no AJAX é inicializada pela criação de um objeto do tipo XMLHttpRequest e a subsequente utilização de seus métodos e propriedades:

  • método open(metodo, url, assincronicidade): por onde são passados o tipo da requisição (GET, POST, DELETE...), a url de onde se quer acessar os recursos e se a requisição deve ser tratada de forma assíncrona ou não;
  • propriedade onload: com uma função de callback atribuída a ela, para lidar com a resposta ou possíveis erros;
  • método send(): para enviar a requisição.

Axios

O Axios é uma biblioteca que facilita as requisições HTTP, podendo trabalhar tanto ser utilizada no front-end quanto no back-end. Ela primeiramente deve ser instalada (utilizando npm, temos o comando npm i axios) e importada no projeto. Ela é uma biblioteca baseada em Promise, de modo que podemos utilizar os blocos .then/.catch para lidar com a resposta ou com possíveis erros, respectivamente.

Um das vantagens do Axios é que ele também oferece atalhos para cada tipo de requisição, como GET, PUT, POST, DELETE, com a notação de ponto. Ou seja, para realizar uma requisição GET temos axios.get(), para uma requisição DELETE temos axios.delete(), entre outros tipos de requisição.
