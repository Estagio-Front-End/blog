Estruturas de condicionais

Condicionais em Javascript, como o nome já diz, são estruturas de condicional em que, caso uma condição seja verdadeira, um bloco de código será executado, caso contrário, a condição sendo falsa, outro bloco de código será executado. Cada estrutura possui sua aplicação, e elas podem ser combinadas entre si, ou seja, uma estrutura de condicional pode ter cada uma das 4 partes mencionadas anteriormente. A única obrigatoriedade existente é que, para que se tenha uma estrutura else, else if ou aninhamento de condicionais, deve-se inicializar a estrutura com um if.

  • Estruturas somente com if testam somente uma expressão de condição (valor dentro dos parêntesis) e explicitam o código a ser executado apenas para o caso de a condição ser verdadeira.
  • Estruturas com if e else testam somente uma expressão de condição e explicitam o código a ser executado para o caso de a condição ser verdadeira (bloco de código dentro do par de chaves após o if) e para o caso de a condição ser falsa (bloco de código dentro do par de chaves após o else). Estruturas condicionais não podem ter mais de um else, portanto, caso seja necessário avaliar outras expressões, é necessário utilizar outras estruturas.
  • Estruturas com if e else if podem testar mais de uma expressão de condição, porém com a obrigatoriedade de que, para que a condição do else if seja avaliada, a condição do if deve ser falsa. Mais de um else if pode ser utilizado, até que todas as expressões a serem avaliadas sejam contempladas. No final deste tipo de estrutura, é possível, também, finalizar com um else, para contemplar casos em que nenhuma das condições avaliadas seja verdadeira.
  • Estruturas com ifs aninhados podem testar mais de uma expressão de condição, porém com a obrigatoriedade de que, para que a condição do if mais interno seja avaliada, a condição do if mais externo deve ser verdadeira. Este tipo de estrutura também pode ser combinada com else if e else, a depender do tipo de condições a serem avaliadas.

Operadores de comparação

Os operadores de comparação são muito utilizados para criar expressões a serem avaliadas. Os principais operadores de comparação são:

  • > avalia se um valor é maior que outro.
  • < avalia se um valor é menor que outro.
  • >= avalia se um valor é maior ou igual a outro.
  • <= avalia se um valor é menor que outro.
  • !== avalia se um valor é diferente de outro.
  • === chamado de igualdade estrita, avalia se um valor é igual a outro. Ese operador é quase sempre a operação de comparação correta a se usar.
  • == chamado de igualdade ampla, avalia se um valor é igual a outro após a conversão dos valores para um tipo comum. Desta forma "10" == 10 geraria um resultado true.

Operadores lógicos

Os operadores lógicos são utilizados para junção de expressões a serem avaliadas, podendo, em alguns casos, substituir aninhamentos de estruturas com if. Os operadores lógicos são:

  • AND - && atua como o conector "e", em que expressões conectadas por este operador devem todas retornar true individualmente para que a expressão seja considerada true.
  • OR - || atua como o conector "ou", em que pelo menos uma das expressões conectadas por este operador deve retornar true individualmente para que a expressão seja considerada true.
  • NOT - ! atua como o operador "não", em que, expressões precedidas por ! terão seu valor negado. Caso a expressão seja true, ela retornará false e vice-versa.

Na condicional a seguir, temos:
if ((x === 5 || y >= 5) && !(z < 10)) {
  //bloco de código
}

Para que o bloco de código seja executado, temos as seguintes condições:

  • - Devido ao operador &&, as expressões (x === 5 || y >= 5) e !(z < 10) devem retornar, individualmente, true.
  • - No primeiro parêntesis, devido ao operador ||, qualquer uma das duas expressões sendo verdadeira, a expressão total será verdadeira.
  • - No segundo parêntesis, devido ao operador !, a expressão z < 10 deve retornar false para que, com o operador de negação, a expressão total retorne true.
