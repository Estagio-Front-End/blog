Regex em Javascript

JAVASCRIPT

As Regex, ou expressões regulares, são formas de criar padrões a serem utilizados, muito comuns em inputs de formulários. Com as regex, pode-se realizar operações de busca, substituição, validação de padrões e filtragem de informações. As regex, em Javascript, podem ser criadas de duas formas. A primeira é de forma literal, atribuindo à uma variável e utilizando barras duplas (//) em volta do padrão criado com as flags após a segunda barra. A segunda, com o construtor RegExp(expressão, flags) atribuído à uma variável.

Caracteres e flags

Com as regex, é possível buscar por caractéres específicos (/c/) ou palavras específicas (/Camila/). Ao confrontar um texto com uma regex, ela poderá buscar apenas pela primeira correspondência ou poderar encontrar ocorrências em todo o texto, isto dependerá das flags. As flags mais utilizadas são:

  • g: significa global, ou seja, serão retornadas todas as correspondências com a regex utilizada.
  • i: significa insensitive. As regex, por padrão, são case sensitive, ou seja, /c/ e /C/ são regex diferentes. Esta flag faz com que se busque por letras maiúsculas e minúsculas por meio de uma mesma regex.
  • m: significa multi line. Com esta flag, a regex buscará por ocorrências em todas as linhas de um texto ou campo, dado que o padrão possua os caractéres ^ e $ indicando início e fim da linha, respectivamente.
  • s: significa single line. Com esta flag, a regex só buscará por ocorrências na primeira linha de um texto ou campo.
  • x: significa extended. Faz com que a regex ignore espaços em branco.

Operador lógico |

As regex também possuem a possibilidade de inclusão de um operador lógico, similar ao OR (||), que é o pipe (|). Deste modo, com um regex /c|a|m/gi, por exemplo, serão procuradas todas as ocorrências das letras c, a ou m, maíusculas ou minúsculas.

Conjuntos "[]" e ranges

Para situações em que se pode ter variados tipos de caractéres em uma mesma casa, são utilizados os conjuntos "[]". Este tipo de sintaxe é muito utilizada em ranges, por exemplo [0-9] para números de 0 à 9 ou [a-z] para letras do alfabeto minúsculas. è possível, também, negar conjuntos com o caractere (^). Desta forma, com um regex /[^0-9]/gi, serão buscadas ocorrências de todos os caracteres diferentes de números de 0 a 9.

Shorthands

Em regex, também existem as shorthands para facilitar a escrita do padrão. Os mais utilizados são \d com o mesmo efeito que [0-9], \w com o mesmo efeito que [a-zA-Z0-9_] e \s para espaços em branco.

Quantificadores

Para quantificar a ocorrência de um caractere ou grupo, é possível utilizar os caracteres especiais * (zero ou mais ocorrências), ? (zero ou uma ocorrência), + (uma ou mais ocorrências) ou a notação {i, f} (de i à f ocorrências). A regex /\d+/gi, por exemplo, buscará grupos com uma ou mais ocorrências de dígitos de 0 a 9.

Início e fim de linha

Para identificar o início e o fim de uma linha e auxiliar a busca de caracteres ou grupos em uma regex, utiliza-se os caracteres especiais ^ para indicar o início da linha (o caractere deve estar fora do grupo ou conjunto, pois dentro seria uma negação) e o $ para indicar o fim de uma linha.

Grupo "()"

É possível criar subgrupos dentro da regex com os parênteses, criando uma "subregra" assim como referências à esses subgrupos. Estas referências podem ser utilizadas na própria regex ou na string de troca do método replace. Em uma regex de CPF, por exemplo: /(\d{3})[\.]?(\d{3})[\.]?(\d{3})[\-]?(\d{2})/g podemos ter 4 grupos: 3 grupos com 3 dígitos e 1 grupo com 2 dígitos. Em uma string de troca, estes grupos poderão ser referidos como $1 a $4.

Métodos de Regex

Para realizar as operações de busca, substituição, validação de padrões e filtragem de informações, as regex possuem métodos como test, replace, entre outros. São eles:

  • regex.exec(item) confronta o item passado como argumento com a regex. Retorna um array com informações como a primeira correspondência, o índice desta correspondência, caso haja, e o item passado como argumento. Caso não haja correspondências, este método retornará null.
  • regex.test(item): confronta o item passado como argumento com a regex. Retorna um booleano, true se houverem correspondências e false caso contrário.
  • item.match(regex): confronta o item com a regex passada como argumento. Retorna um array com todas as correspondências encontradas, ou null caso não haja correspondências.
  • item.replace(regex, item2): confronta o item com a regex passada como argumento. Em seguida, troca todas as correspondências encontradas pelo item passado como segundo argumento, seja ele uma string ou conformando o valor ao padrão da regex.
  • item.search(regex): confronta o item com a regex passada como argumento. Retorna o índice da primeira correspondência encontrada ou -1 caso não haja correspondências.
  • item.split(regex): confronta o item com a regex passada como argumento. Quebra o item a cada correspondência encontrada, gerando um array em que os itens são o "resto" do item original sem a parte que teve correspondência com o regex. Caso não haja correspondências, retorna um array de 1 item: o item original.

Máscaras

Máscaras em inputs são utilizadas para mostrar ao usuário que ele escreveu informações requisitadas de forma correta, ou mesmo mostrar a forma como as informações devem ser escritas. As máscaras são criadas com base nas regex, visto que estas expressões são os padrões que indicam a informação escrita de forma correta.
