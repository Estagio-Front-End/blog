Design Responsivo

RESPONSIVIDADE

Na época dos primeiros websites, diferentes dimensões de telas não eram levadas em consideração no desenvolvimento pois essa diversidade não era uma realidade presente. Os sites eram desenvolvidos para um tamanho específico, geralmente de desktop, e telas maiores ou menores sofreriam as consequências na visualização. Com a popularização dos dispositivos móveis, se tornou cada vez mais imperativo que os sites pudessem se adequar à tela do usuário. No Brasil, em 2019, segundo fontes do ThinkWithGoogle, 58% dos brasileiros acessavam a internet exclusivamente pelo celular e 85% das pessoas das classes D e E acessavam a internet via celular. Apenas com estes números, é possível ver a grande importância de se pensar em dispositivos móveis no desenvolvimento.

Diante disto, as técnicas de design e desenvolvimento foram se aprimorando para abarcar as novas necessidades. Primeiramente, existiam os sites com largura líquida, técnica que resultava em conteúdos se sobrepondo em telas menores e linhas muito grandes em telas maiores. Outra solução era desenvolver 2 sites com urls diferentes, uma para desktop e uma para mobile, o que suscitava a necessidade de desenvolver e manter 2 sites. Em 2004, Cameron Adams descreveu um método para criar designs adaptáveis, mas que necessitava de Javascript para detectar a resolução da tela e carregar o CSS adequado.

O design responsivo, termo cunhado por Ethan Marcotte em 2010, surgiu como solução para a crescente necessidade de sites responsivos, combinando 3 técnicas para o desenvolvimento de sites completamente adaptáveis à tela do usuário.

  • Grids fluidas: grids fluidas geram sites em que não é necessária uma marcação para cada tamanho de tela, mas sim, o layout é adaptável a ponto de se adequar automaticamente ao dispositivo de onde se acessa o site. Atualmente, os modelos de flexbox e grid são os mais utilizados para atingir esse objetivo.
  • Imagens fluidas: esta técnica combina a definição de max-width: 100% para tags do tipo img, de modo que imagens maiores sempre serão reduzidas ao tamanho total da tela, com a utilização da tag picture e os atributos srcset e size da tag img. A utilização desse conjunto permite incluir várias resoluções de uma mesma imagem que se adequem a cada tamanho de tela, de forma que o usuário não precise baixar uma imagem muito maior do que o tamanho que será visualizado.
  • Media queries: este recurso do CSS permite que blocos de código de estilização sejam aplicados de acordo com regras especificadas. Desta forma, é possível, para cada tamanho de dispositivo, criar blocos de código que adequem o layout para aquele dispositivo em específico. Mais sobre este tema no artigo Media Queries.

Um tema que não foi abordado inicialmente, mas também é de extrema importância para designs responsivos é o de tipografia responsiva. Nela, o tamanho da fonte se adequa ao tamanho da tela, diminuindo para telas menores e aumentando para maiores. Este tema se utiliza muito das unidades de medida vw e em/rem, além da função calc(), para setar tamanhos mínimos e responsivos em uma só linha.

Existem, dentro do design responsivo, duas abordagens de escalonamento de layout mais utilizadas: a desktop first e a mobile first. Na primeira, o layout inicial parte do tamanho de um desktop e as media queries são utilizadas diminuir o layout de acordo com os breakpoints que serão utilizados. Na segunda, o layout inicial parte do tamanho de dispositivo móvel, em geral celulares, e as media queries são utilizadas para escalonar o layout para dispositivos maiores como tablets, desktops e monitores full HD. Ambas possuem suas vantagens e desvantagens, de modo que é necessário considerar o público alvo do site, o tipo de conteúdo e os recursos de desenvolvimento disponível para tomar uma decisão de que abordagem seguir.
