Flexbox

POSICIONAMENTO

O flexbox, nome advindo da junção das palavras Flexible Box, é um modelo de posicionamento para layouts que permite maior flexibilidade de posicionamento de itens e facilita as adaptações do layout para garantir a responsividade, quando comparado com modelos como o position ou o float. Este modelo possui alguns conceitos iniciais nos quais suas propriedades irão se basear para gerar o comportamento esperado. São estes:

  • Flex container: é o elemento cujos itens irão ter o comportamento flexível.
  • Flex item: é um item dentro de um container flexível.
  • Eixo principal: eixo que definirá de onde o posicionamento dos itens iniciará. A depender da propriedade flex-direction, pode ser o eixo vertical ou horizontal. A propriedade na sua forma default tem o eixo horizontal como principal.
  • Eixo secundário: eixo que é secundário ao eixo principal. A depender da propriedade flex-direction, pode ser o eixo vertical ou horizontal. A propriedade na sua forma default tem o eixo vertical como secundário.
  • main-start / main-end: início e fim do eixo principal, seja ele vertical ou horizontal, a depender da propriedade flex-direction.
  • cross-start / cross-end: início e fim do eixo secundário, seja ele vertical ou horizontal, a depender da propriedade flex-direction.
  • main size / cross size: tamanhos dos eixos principal e secundário, sejam eles o vertical ou horizontal, a depender da propriedade flex-direction.

Para inicializar um flex container, deve ser inclusa a propriedade e valor display: flex em seu bloco de código. Dessa forma, os itens contidos neste flex container se tornarão flex items e as propriedades do flexbox poderão ser aplicadas. As propriedades são:

  • flex-direction: define qual o eixo principal do container, horizontal (row) ou vertical (column). Se o eixo horizontal é o principal, automaticamente o eixo vertical se torna o secundário e vice-versa. Há também os valores row-reverse e column-reverse, que definem, além do eixo principal, que a ordem de posicionamento dos itens será inversa à sua ordem no DOM.
  • justify-content: esta propriedade define como os elementos serão justificados no seu eixo principal. Pode ter valores como center, flex-start, flex-end, space-around, space-between e space-evenly.
  • align-items: esta propriedade define como os elementos serão alinhados no seu eixo secundário. Pode ter valores como center, flex-start, flex-end e stretch.
  • align-content: esta propriedade define o espaçamento entre os flex items no seu eixo secundário. Pode ter valores como center, flex-start, flex-end, stretch, space-around e space-between.
  • flex-wrap: esta propriedade define se, caso a soma dos tamanhos dos itens ultrapasse o main size disponível, se os itens poderão ou não ser quebrados em novas linhas/colunas. Possui valores de wrap (pode quebrar), nowrap (não pode quebrar) e wrap-reverse (pode quebrar e os elementos são posicionados na ordem reversa ao normal).
  • flex-flow: é um shorthand que agrupa as propriedades de flex-direction e flex-wrap, nesta ordem. Deste modo, seus valores possíveis são as combinações entre os valores destas duas propriedades.
  • gap: define um valor de espaçamento entre flex items, assumindo valores numéricos com unidade. Caso sejam definidos 2 valores, como em gap: 10px 30px, o primeiro valor é o espaçamento entre ramificações do eixo principal e, o segundo, entre ramificações do eixo secundário.
  • align-self: esta propriedade, quando aplicada no bloco de código de estilização de um flex item, sobrescreve a propriedade align-items para aquele item em específico. Deste modo, possui a mesma função e valores possíveis de align-items, mas só é aplicada naqueles que possuírem esta propriedade no código.
  • flex-grow: aplicada no flex item, esta propriedade define um fator de crescimento de um flex item, especificando quanto de espaço dentro do flex container esse item irá ocupar em relação aos demais items. Esta propriedade assume valores numéricos sem unidade e seu valor padrão é 0.
  • flex-shrink: é uma propriedade análoga ao flex-grow, porém, por sua vez, define um fator de encolhimento.
  • flex-basis: esta propriedade define o valor base para a dimensão no eixo principal de um flex item, antes de qualquer ação das propriedades flex-wrap, flex-grow ou flex-shrink, ou seja, antes que o espaço remanescente do container seja distribuído.
  • flex: é um shorthand que agrupa as propriedades de flex-grow, flex-shrink e flex-basis, nesta ordem, sendo as duas últimas opcionais. Deste modo, seus valores possíveis são as combinações entre os valores destas três propriedades.
  • order: aplicada nos flex items, é uma propriedade que define a ordem na qual os elementos serão renderizados. Recebe valores numéricos e sem unidade, indicando, de forma crescente, a ordem de cada elemento ou uma priorização de um elemento específico. Vale ressaltar que a ordem dos elementos no DOM não é alterada por esta propriedade, apenas a ordem de renderização.

As propriedades supracitadas podem ser utilizadas a fim de montar layouts flexíveis e responsivos. É importante pontuar que, já que o flexbox é um modelo com suas próprias propriedades, as propriedades float, clear e vertical-align não têm efeito em flex items. Além disso, somente a hierarquia pai-filho é visualizada no CSS, de modo que itens "netos" não possuirão propriedades flex aplicadas para elementos filhos. Para tal, é necessário que o elemento filho no qual o elemento neto está contido também possua display: flex e as propriedades a serem aplicadas em seu bloco de código.
