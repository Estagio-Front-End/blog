Rotas em Angular

ANGULAR

Rotas em Angular são a ferramenta que possibilita que o usuário navegue por diferentes páginas da aplicação com mais fluidez, sem constantes recarregamentos da página. Esta ferramenta torna a aplicação uma SPA (Single Page Application), em que se pareiam rotas e componentes, renderizando um componente em específico a depender da rota acessada pelo usuário. As rotas são importantes para aumentar a performance da aplicação, possibilitando a utilização de lazy loading, além de facilitar a renderização com base em permissão com as guardas.

Quando se cria uma nova aplicação com o Angular CLI, um arquivo para rotas será criado automaticamente: o app.routes.ts ou, caso a aplicação seja module-based, o app-routing.module.ts, onde será criada grande parte do código que envolve o roteamento da aplicação.

Para a criação das rotas, têm-se alguns passos essenciais: importar rotas no app.config.ts; criar um array de Routes; definir as rotas da aplicação e adicionar as rotas na aplicação. Os dois primeiros passos são feitos automaticamente pelo Angular CLI, restando ao desenvolvedor definir as rotas e componentes dentro do array de Routes e adicioná-las na aplicação. Cada rota é um objeto com duas propriedades: a rota a ser acessada e o componente a ser renderizado.

Para adicionar na aplicação as rotas definidas, é necessário utilizar o seletor router-outlet onde se deseja renderizar o componente definido em cada rota, além do atributo routerLink em âncoras (substituindo o href) para possibilitar a navegação para o usuário.

Lazy loading

O lazy loading é um conceito que auxilia no aumento da performance da aplicação por meio de carregamento apenas daquilo que é necessário, ou seja, o componente associado a uma rota é carregado apenas quando aquela rota em específico é acionada. Como, anteriormente, as aplicações eram carregadas como um todo, fazendo com que esse processo pudesse demorar a depender do tamanho da aplicação, o lazy loading vem como uma forma de encapsular o carregamento, deixando esse processo mais eficiente.

Para a criação da estrutura de rotas com lazy loading, é minimamente necessário um componente, um módulo para o componente, um módulo de roteamento para o componente e o módulo de roteamento da aplicação. O componente é criado e importado no módulo do componente (definido em declarations), o módulo de roteamento é criado e importado no módulo de componente (definido em imports) e, desta forma, a conexão entre os componentes e módulos para o roteamento está completo. O último passo é adicionar a sintaxe de lazy loading do módulo de roteamento da aplicação. Portanto, em resumo:

  • Os componentes necessários são criados e, em seguida, importados e declarados dentro do módulo do componente (modulo-principal.module.ts na propriedade declarations do decorator @NgModules).
  • O módulo de roteamento é criado e, em seguida, importado e declarado dentro do módulo do componente (modulo-principal.module.ts na propriedade imports do decorator @NgModules). Neste módulo de roteamento, é onde ficará o array de Routes contendo as rotas possíveis dentro daquele componente, com a sintaxe usual das rotas (pares path/component).
  • O módulo principal do componente, contendo os componentes e o módulo de roteamento importados, será carregado conforme acessado pelas rotas da aplicação.
  • No componente de roteamento da aplicação, é incluída a sintaxe de lazy loading para carregamento do módulo principal do componente, que, por sua vez, terá toda a lógica de roteamento interno do component.

Rota coringa

A aplicação deve saber lidar com usuários tentando acessar caminhos não existentes da aplicação. Para tal, existe a possibilidade de criar uma rota coringa com o path **, em que, qualquer caminho não existente que o usuário tente acessar, aparecerá um componente criado para lidar específicamente com esta situação.

Permissividade com guards

Para lidar com permissões de acesso em diferentes partes da aplicação, se utilizam as chamadas guards de rotas. Para criação dessas guards, tem-se o comando ng generate guard nome-guard, em que nome-guard é um nome arbitrário. Neste arquivo, são definidas as funções de guard que se deseja utilizar, sendo uma das mais utilizadas a do tipo canActivate, que media permissões para entrar em uma rota.

Criada a função, é necessário adicioná-la como propriedade na definição das rotas, sejam elas dentro do um componente ou na aplicação como um todo.

Portanto, com as funcionalidades citadas anteriormente, é possível criar uma navegação personalizada para a aplicação com melhor performance e mais segura em termos de permissão de usuários.
