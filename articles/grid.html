Grid

POSICIONAMENTO

O display Grid é um modelo de posicionamento que traz mais flexibilidade no posicionamento de itens na tela. Este modelo simula uma "grade", com linhas, colunas e células, onde os itens serão posicionados. É considerado por alguns como uma melhoria do Flexbox, visto que traz consigo algumas das vantagens deste modelo e consegue, apenas com suas proprias propriedades, posicionar itens que, no outro modelo, teriam que ser envoltos por elementos div no HTML para que o posicionamento ocorresse da forma esperada. Dessa forma, com o Grid, é possível reduzir o tamanho do código HTML em relação à layouts utilizando Flexbox.

Este modelo possui alguns conceitos iniciais para que seu entendimento seja mais facilitado na hora da aplicação das propriedades. São estes:

  • grid container: definido pela propriedade display: grid em seu bloco de código, é o elemento pai cujos elementos serão posicionados conforme o modelo Grid.
  • grid item: todos os elementos que são filhos diretos de um grid container tornam-se, automaticamente, grid items.
  • grid tracks: na definição do grid, são criadas linhas e colunas. Um grid track é o espaço entre duas grid lines dentro do grid.
  • grid lines: as grid lines são as linhas dentro do template grid. É importante não confundir este conceito com o de grid column, pois as linhas são as separações entre os elementos.
  • grid column: é uma coluna dentro do grid template.
  • grid row: é uma linha dentro do grid template.
  • grid area: é uma área do grid template, que deve sempre ser retangular.
  • fr: é uma unidade de medida que significa unidade fracional de espaço disponível. Se criamos uma grid com 3 colunas de 1fr cada, temos que 33,33% do espaço disponível para o grid será destinado a cada coluna, que aumenta ou diminui de tamanho a depender do tamanho total disponível.
  • minmax(): é uma forma de notação que define os tamanho mínimo e máximo de uma linha ou coluna do grid template. Se temos, por exemplo, uma coluna com minmax(200px, 1fr) isso significa que ela terá, no mínimo, 200px e, a partir disso, crescerá conforme o espaço disponível para o grid.
  • repeat(): é uma forma de deixar o código mais conciso quando temos repetição de tamanhos de coluna ou linhas. Podemos, ao escrever que teremos 3 linhas, cada uma com 1fr de tamanho, utilizar o valor repeat(3, 1fr). Inicia-se com quantas vezes o valor irá se repetir e, depois, qual será a dimensão.

Propriedades de posicionamento e tamanho

  • grid-template-columns: define a quantidade e o tamanho das colunas dentro de um grid template. Caso o número de linhas não seja definido, gerando um grid implícito os itens se acomodarão em linhas respeitando o espaço disponível e o número de colunas.
  • grid-template-rows: define a quantidade e o tamanho das linhas dentro de um grid template. Quando utilizada junto à propriedade grid-template-columns, gera um grid explícito, com linhas e colunas definidas.

  • grid-row: define em quais grid lines do eixo das linhas o elemento inicia e termina, atuando como shorthand das propriedades grid-row-start e grid-row-end. Os valores assumidos são numéricos sem unidades ou pode ser utilizada a palavra reservada span, sendo os dois valores separados pelo símbolo /. Nesse contexto de grid, o span representa "quantas grid lines à frente de onde o elemento inicia". Desta forma, por exemplo, se temos um elemento que inicia na grid line 1 e aplicamos span 2 como segundo valor do seu grid-row, o elemento terminará na grid line 3, ocupando as linhas 1 e 2.
  • grid-column: define em quais grid lines do eixo das colunas o elemento inicia e termina, atuando como shorthand das propriedades grid-column-start e grid-column-end. Os valores assumidos são numéricos sem unidades ou pode ser utilizada a palavra reservada span, sendo os dois valores separados pelo símbolo /.

  • order: esta propriedade permite o controle da ordem na qual os elementos são posicionados dentro do grid. Assume valores numéricos sem unidade e, quanto menor o número, maior a prioridade de posicionamento do item. Assim como no modelo flexbox, esta propriedade não altera a ordem dos elementos no DOM, apenas na exibição.

Propriedades de alinhamento

  • justify-content: esta propriedade alinha o bloco de conteúdo de um grid container ao longo do eixo das linhas, quando o tamanho do conteúdo não preenche todo o espaço disponível. Seu valor padrão é stretch, mas pode assumir valores como center, start, end, space-between, space-around e space-evenly.

  • place-content: shorthand para as propriedades align-content e justify-content, nesta ordem. Deste modo, seus valores possíveis são as combinações dos valores das duas propriedades agrupadas. Caso se use apenas um valor, este será aplicado para ambas as propriedades.

  • Com os conceitos de todos os alinhamentos definidos, é possível diferenciá-los por: propriedades com content alinham tudo o que há dentro do grid container como se fosse um elemento só e devem ser aplicadas no bloco de código do grid container. Propriedades com items alinham os items do grid container em relação à célula do grid que ocupam e devem ser aplicadas no bloco de código do grid container por afetarem todos os itens existentes no grid. Propriedades com self são similares às propriedades com items, porém, devem ser aplicadas no bloco de código do grid item e seus valores são reproduzidos apenas no item no qual o código foi aplicado.
