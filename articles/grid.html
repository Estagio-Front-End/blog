Grid

POSICIONAMENTO

O display Grid é um modelo de posicionamento que, assim como o Flexbox, traz mais flexibilidade no posicionamento de itens na tela. Este modelo simula uma "grade", com linhas e colunas, onde os itens serão posicionados. É considerado por alguns como uma melhoria do flexbox, visto que traz consigo algumas das vantagens deste modelo e consegue, apenas com suas proprias propriedades, posicionar itens que, no Flexbox, teriam que ser envoltos por elementos div no HTML para que o posicionamento ocorresse da forma esperada. Dessa forma, com o Grid, é possível reduzir o tamanho do código HTML em relação à layouts utilizando Flexbox.

Este modelo possui alguns conceitos iniciais para que seu entendimento seja mais facilitado na hora da aplicação das propriedades. São estes:

  • grid container: definido pela propriedade display: grid em seu bloco de código, é o elemento pai cujos elementos serão posicionados conforme o modelo Grid.
  • grid item: todos os elementos que são filhos diretos de um grid container tornam-se, automaticamente, grid items.
  • grid tracks: na definição do grid, são criadas linhas e colunas. Um grid track é o espaço entre duas grid lines dentro do grid.
  • grid lines: as grid lines são as linhas dentro do template grid. É importante não confundir este conceito com o de grid column, pois as linhas são as separações entre os elementos.
  • grid column: é uma coluna dentro do grid template.
  • grid row: é uma linha dentro do grid template.
  • grid area: é uma área do grid template, que deve sempre ser retangular.
  • fr: é uma unidade de medida que significa unidade fracional de espaço disponível. Se criamos uma grid com 3 colunas de 1fr cada, temos que 33,33% do espaço disponível para o grid será destinado a cada coluna, que aumenta ou diminui de tamanho a depender do tamanho total disponível.
  • minmax(): é uma forma de notação que define os tamanho mínimo e máximo de uma linha ou coluna do grid template. Se temos, por exemplo, uma coluna com minmax(200px, 1fr) isso significa que ela terá, no mínimo, 200px e, a partir disso, crescerá conforme o espaço disponível para o grid.
  • repeat(): é uma forma de deixar o código mais conciso quando temos repetição de tamanhos de coluna ou linhas. Podemos, ao escrever que teremos 3 linhas, cada uma com 1fr de tamanho utilizar o valor repeat(3, 1fr). Inicia-se com quantas vezes o valor irá se repetir e, depois, qual será a dimensão.

Definidos os conceitos iniciais e utilizando o código display: grid para inicializar um grid container, é possível utilizar as propriedades inerentes a este modelo, que são:

  • grid-template-columns: define a quantidade e o tamanho das colunas dentro de um grid template. Caso o número de linhas não seja definido, gerando um grid implícito os itens se acomodarão em linhas respeitando o espaço disponível e o número de colunas.
  • grid-template-rows: define a quantidade e o tamanho das linhas dentro de um grid template. Quando utilizada junto à propriedade grid-template-columns, gera um grid explícito, com linhas e colunas definidas.
  • grid-template-areas: define áreas nomeadas dentro de um grid template. É uma forma mais intuitiva de definir onde cada elemento será posicionado. É utilizado em conjunto com a propriedade grid-area.
  • grid-area: aplicada nos grid items, define um nome para o elemento para que ele seja intuitivamente posicionado no grid-template-areas. Esta propriedade também pode ter o comportamento de shorthand para outras propriedades, mas isto será abordado posteriormente.
  • grid-template: shorthand que agrupa as propriedades grid-template-columns, grid-template-rows e grid-template-areas. Os tamanhos de linhas são definidos ao lado da definição de cada área de linha e os tamanhos das colunas são definidos no final, separados por um símbolo / do restante do código da propriedade.
  • grid-row-gap / row-gap: esta propriedade cria um espaçamento entre linhas de um grid. Pode receber valores numéricos com unidade ou em porcentagem.
  • grid-column-gap / column-gap: esta propriedade cria um espaçamento entre colunas de um grid. Pode receber valores numéricos com unidade ou em porcentagem.
  • grid-gap / gap: shorthand que agrupa as propriedades grid-row-gap e grid-column-gap. Quando utilizado 1 valor, este será aplicado para ambas as propriedades. Quando utilizados 2 valores distintos, o primeiro será aplicado no espaçamento entre linhas e o segundo, entre colunas.
  • grid-auto-columns: define o tamanho de colunas criadas implicitamente, ou seja, quando há mais elementos do que o definido no grid template. Pode receber valores numéricos com unidade, porcentagens ou unidades fracionais.
  • grid-auto-rows: define o tamanho de linhas criadas implicitamente, ou seja, quando há mais elementos do que o definido no grid template. Pode receber valores numéricos com unidade, porcentagens ou unidades fracionais.
  • grid-auto-flow: define como elementos cujo posicionamento não está definido irão popular o grid, estejam eles em linhas/colunas explícitas ou implícitas. Pode receber os valores row, column, column dense ou row dense. O valor dense indica ao algoritmo de posicionamento que tente preencher espaços vazios, de modo que pode deixar o layout desordenado, de forma que seu uso não é muito recomendado.
  • grid-column: define em quais grid lines do eixo das colunas o elemento inicia e termina, atuando como shorthand das propriedades grid-column-start e grid-column-end. Os valores são numéricos sem unidades ou pode ser utilizada a palavra reservada span, sendo os dois valores separados pelo símbolo /. Nesse contexto de grid, o span representa "quantas grid lines à frente de onde o elemento inicia". Desta forma, por exemplo, se temos um elemento que inicia na grid line 1 e aplicamos span 2 como segundo valor do seu grid-column, o elemento terminará na grid line 3, ocupando as colunas 1 e 2.
  • grid-row: define em quais grid lines do eixo das linhas o elemento inicia e termina, atuando como shorthand das propriedades grid-row-start e grid-row-end. Os valores são numéricos sem unidades ou pode ser utilizada a palavra reservada span, sendo os dois valores separados pelo símbolo /.
