Local storage e Session storage

Integrantes da Web Storage API, o localStorage e o sessionStorage são formas mais intuitivas de armazenar dados no navegador em comparação aos Cookies. Ambos armazenam dados na forma chave/valor e estes dados podem ser recuperados e reutilizados na aplicação. A persistência dos dados em ambos os recursos é por origem, portanto, protocolos diferentes (http ou https), domínios diferentes ou portas diferentes terão localStorage e sessionStorage diferentes. As grandes diferenças entre os dois são:

  • Compartilhamento entre abas e janelas: o localStorage compartilha dados com outras abas ou janelas na mesma origem (mesmos protocolo, domínio e porta). O sessionStorage não faz esse compartilhamento.
  • Expiração dos dados no navegador: os dados salvos no sessionStorage duram apenas até o usuário fechar a aba ou janela. Dados salvos no localStorage irão persistir mesmo que o navegador seja fechado e aberto novamente, podendo os dados ser removidos apenas por meio de código Javascript ou pela limpeza do cache do navegador.

Para ambos, existem métodos de manipulação dos dados. São eles:

  • (local/session)Storage.setItem(chave, valor): insere um par chave/valor para armazenamento no objeto Storage indicado. É importante frisar que ambos aceitam apenas chaves e valores do tipo string, de modo que quaisquer outros tipos devem ser transformados em string.
  • (local/session)Storage.key(índice): retorna a chave do item armazenado no objeto Storage no índice indicado como argumento.
  • (local/session)Storage.getItem(chave): retorna o valor que foi armazenado como par da chave indicada como argumento, sempre em formato de string.
  • (local/session)Storage.length: propriedade que retorna o número de pares chave/valor armazenados no web storage indicado.
  • (local/session)Storage.removeItem(chave): remove do web storage indicado o par chave/valor identificado pela chave passada como argumento.
  • (local/session)Storage.clear(): deleta todo o conteúdo do web storage indicado.

Todos os métodos da Web Storage API são síncronos, de modo que, a depender da quantidade de dados armazenados, a performance da aplicação pode ser afetada. Desta forma, para grandes volumes de dados, é indicado buscar outros recursos de armazenamento.

Cookies

Os cookies são uma forma de armazenamento de dados em texto que servem para "melhorar a experiência do usuário", guardando preferências de sistema, dados de sessão ou informações sobre o comportamento de um usuário em uma página na web. Como os cookies podem guardar informações sensíveis, legislações mais atuais definiram que os sites precisam pedir a permissão do usuário para coletar e armazenar estas informações.

Os cookies podem ser definidos adicionando uma linha no header em uma requisição HTTP com o Set-Cookie ou, mais facilmente, com o document.cookie. Em ambos, são admitidos pares chave=valor separados por ponto e vírgula mais um espaço. Nesses pares, podem aparecer atributos como Domain com o valor do host para o qual o cookie será enviado, Path com o valor de caminho que deve estar na url, Expire com a data e hora de expiração do cookie, entre outros.

Os cookies, de acordo com persistência, podem ser de dois tipos, de sessão ou permanentes. Os nomes são auto explicativos, mas, em código, a diferença entre eles é que os cookies permanentes possuem atributos Expire ou MaxAge definidos, ao passo que os cookies de sessão não possuem estes atributos definidos e expiram ao término da sessão no navegador.

Em termos de escopo, os cookies podem ser de first-party (cookies primários), second-party ou third-party (cookies de terceiros).

  • First-party: um cookie enviado diretamente pelo site que o usuário está utilizando. Por exemplo: o carrinho de compras de um e-commerce.
  • Second-party: um cookie compartilhado com outra parte por meio de um contrato de parceria de dados. Por exemplo: relatório de uso do Microsoft Teams.
  • Third-party: um cookie instalado por um site diferente do qual o usuário está acessando. Por exemplo: botões de curtida de redes sociais.

Na seção "Application", aba "Cookies" do Chrome DevTools, é possível visualizar os cookies que estão sendo utilizados no site em que se está visitando.

Teste a seguir a funcionalidade do localStorage e do sessionStorage. Roupas adicionadas ao carrinho serão salvas no localStorage e sapatos, no sessionStorage.
